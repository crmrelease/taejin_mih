# 작업 규칙 (taejin_mih)

이 repo는 세 Claude agent 워크스페이스(som, rang, mo)가 공유한다.

## 워크스페이스별 작업 브랜치
- `agent_rang` 워크스페이스 → `rang` 브랜치에서만 작업
- `agent_som` 워크스페이스 → `som` 브랜치에서만 작업
- `agent_mo` 워크스페이스 → `mo` 브랜치에서만 작업
- `main`은 통합 대상. agent들은 직접 푸시하지 않고 PR/사용자 검토를 거쳐 머지된다.

## 응답 스타일
- 사용자에게 응답할 때는 **존댓말**을 사용한다. 반말은 사용하지 않는다.

## 토큰 관리
- GitHub 토큰은 각 워크스페이스의 `.env`에서 관리하며 절대 커밋하지 않는다 (`.gitignore`로 차단).
- 푸시할 때 `origin` URL에 토큰을 박지 않고, `.env`를 source한 뒤 inline tokenized URL을 사용한다.
- `ANTHROPIC_API_KEY`는 절대 설정하지 않는다. Max 플랜 인증만 사용한다.

## 봇 필수 규칙 (2026-05-11 사고 기반)

아래 규칙은 실제 운영 장애에서 도출되었으며, 모든 봇 코드 수정 시 반드시 준수한다.

### 1. 시작 시 신원 검증 (필수)
- 봇 시작 시 `auth.test`로 받은 `userId`가 자기 것이 맞는지 검증한다.
- `.env`에 `EXPECTED_USER_ID`를 명시하고, 불일치 시 **시작을 거부**한다.
- 잘못된 토큰으로 registry에 등록되는 것을 방지한다.
```
사고: rang이 mo의 토큰으로 시작 → mo의 userId로 registry 등록 → 메시지 중복 처리
```

### 2. 프로세스 중복 방지 (필수)
- 시작 시 PID lock file을 생성한다 (`bot/.pid`).
- 이미 같은 봇이 실행 중이면 **시작을 거부**한다.
- 종료 시 lock file을 삭제한다.
```
사고: rang이 2개 동시에 실행 → 좀비 프로세스 발생
```

### 3. claude CLI 호출 규칙 (필수)
- `spawn` 시 stdin은 반드시 `ignore`로 설정한다: `stdio: ['ignore', 'pipe', 'pipe']`
- stdin을 열어두면 claude가 입력 대기 → 3초 후 경고 → exit 1 발생한다.
```
사고: stdin 미처리 → "Warning: no stdin data received in 3s" → claude exited 1 반복
```

### 4. 실패 시 backoff (필수)
- claude CLI가 exit 1을 반환하면 즉시 재시도하지 않는다.
- rate limit / 사용량 초과 시 **지수 backoff** 적용 (30초 → 1분 → 2분 → 최대 10분).
- 연속 3회 실패 시 슬랙에 상태 알림을 보낸다: "사용량 한도 도달, 복구 대기 중"
- 사용자에게는 리액션(⏳)으로 "지금 응답 불가" 상태를 표시한다.
```
사고: 사용량 초과 → claude 반복 호출 → 크래시 루프 → launchd 재시작 반복
```

### 5. Slack 메시지 길이 제한 (필수)
- 응답을 슬랙에 보내기 전에 **3500자**로 truncate한다 (마크다운 오버헤드 감안).
- 초과 시 "...(전체 응답은 스레드에서 분할)" 처리한다.
```
사고: som 봇 msg_too_long 에러
```

### 6. 봇 자체 health check (필수)
- 시작 완료 시 슬랙에 "✅ [봇이름] 시작됨" 알림을 보낸다.
- 주기적으로 heartbeat를 로그에 기록한다 (5분마다).
- 비정상 종료 시 원인을 로그에 남긴다.
```
사고: 봇이 죽어도 아무도 모름 → 사용자가 직접 확인해야 알 수 있었음
```

### 7. 모니터 범위 확장 (권장)
- monitor.ts는 외부 서비스(JSONBin, 페이지, imgbb)만 감시하지 말고, **봇 프로세스 상태도 감시**한다.
- 봇이 응답 불능이면 슬랙 알림을 보낸다.

### 8. launchd 크래시 루프 대응 (필수)
- 연속 실패 시 무한 재시작되지 않도록 봇 내부에 **시작 실패 카운터**를 둔다.
- 5회 연속 시작 실패 시 자체 종료하고, 슬랙에 "수동 개입 필요" 알림을 보낸다.
- 상태 파일에 마지막 실패 시각을 기록하여 크래시 루프를 감지한다.
```
사고: PC 재시작 → 사용량 초과 상태에서 launchd가 무한 재시작 → 상태 꼬임
```

## 코드 수정 시 일반 규칙

### 방어적 프로그래밍
- 외부 입력(환경변수, 토큰, API 응답)은 사용 전에 반드시 검증한다.
- "정상 동작"만 구현하지 말고, "실패 시 어떻게 되는가"를 항상 고려한다.
- 새 기능 추가 전에 기존 안정성 이슈를 먼저 해결한다.

### 공유 자원 접근
- registry(`/Users/somi/Desktop/mih/.agents/`)는 공유 디렉토리다. 쓰기 전에 자기 것만 수정한다.
- 다른 봇의 파일을 읽는 것은 허용하되, 수정은 금지한다.
- 파일 쓰기 시 atomic write(임시 파일 → rename)를 사용한다.
