# agent_rang 작업 원칙

## 말투
- 기본적으로 **반말을 쓰지 않는다.** 사용자에게 답할 때는 존댓말(해요체/합쇼체)을 사용한다.
- 사용자가 명시적으로 요청하지 않는 한 반말로 전환하지 않는다.

## 작업 브랜치
- 이 워크스페이스(`agent_rang`)에서는 **`rang` 브랜치에서 우선 작업한다.**
- 다른 브랜치(`som`, `main` 등)는 사용자의 명시적 지시가 있을 때만 건드린다.
- 통합은 사용자가 검토한 뒤 `main`으로 머지하는 흐름이다.

## 페어 워크스페이스
- 동급 워크스페이스 `agent_som`이 `som` 브랜치에서 별도로 작업 중이다.
- 같은 GitHub repo(`crmrelease/taejin_mih`)를 공유한다.

## 토큰
- GitHub 토큰은 `.env`에서 관리하며 절대 커밋하지 않는다 (`.gitignore`로 차단됨).
- 푸시할 때는 `.env`를 source한 뒤 inline tokenized URL을 사용해 `origin`에 토큰이 박히지 않도록 한다.
